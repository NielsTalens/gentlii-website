# Foundations Diagram Design

**Date:** 2026-04-17

**Goal:** Redesign `foundations.svg` so the left side shows flat input documents, the center is a Gentlii Foundations application screen, and the right side shows a tidy set of organized output documents connected by dotted information-flow lines.

## Approved Direction

Combine:

- Option 1: flat editorial diagram
- Option 3: softer product-illustration feel

## Composition

- Left side contains four flat input documents:
  - `Brief`
  - `Notes`
  - `Research`
  - `Email`
- Center contains a `Gentlii Foundations` application / terminal-style screen.
- Right side contains five aligned output documents:
  - `Strategy`
  - `Business Case`
  - `Product Vision`
  - `Jobs To Be Done`
  - `Product Charter`

## Visual Rules

- Use flat design throughout.
- Keep the existing dark navy palette as the base.
- Use lighter paper surfaces and subtle layering so document cards read clearly.
- Make the left side slightly more varied and informal.
- Make the right side more uniform, aligned, and structured.

## Information Flow

- Input connectors from the left must be dotted and read as information flowing into the application screen.
- Output connectors from the center to the right must also be dotted.
- Lines should be gently curved or routed, not heavy straight bars.
- The flow should visually communicate transformation from fragmented inputs to structured outputs.

## Document Treatment

- Documents should look like files, not simple rectangles.
- Include a small file-tab, folded-corner, or header-strip detail.
- Include a title label and simple internal text lines.

## Refinement Update

- The right side should not use overlapping or stacked file documents.
- The output should instead be shown as five neatly aligned rounded cards.
- These cards should visually relate to the central application's `Organizing` panel.
- The left-side input documents should keep their document form, but the internal text lines should be thicker and softer.
- The left-side input documents should be smaller than before.
- Add a few duplicate offset papers behind the visible input files so the source side feels lightly unorganized.
- Keep the disorder controlled: small offsets, mild overlap, readable front labels, no heavy visual clutter.
- The incoming left-side documents should each connect to the `Gentlii Foundations` intake circle with dotted lines.
- The right side should use one single dotted outbound line with an arrow pointing toward the aligned output cards.

## Success Criteria

- The center no longer looks like a plain block and instead reads as software.
- Left documents read as incoming source material.
- Right documents read as organized deliverables.
- Both connector groups clearly suggest dotted information flow.
